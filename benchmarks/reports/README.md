# Benchmark Reports

This directory holds machine-readable JSON reports emitted by the benchmark and broker-eval harnesses.

## Contents

| File | Purpose |
|------|---------|
| `*-latest.json` | Stable pointer to the most recent run for a suite |
| `*-report-YYYYMMDD-HHMMSS.json` | Timestamped historical report for longitudinal tracking |

## Conventions

- Reports are generated by `scripts/setup/run-grounding-benchmarks.ps1` and `scripts/setup/run-broker-evals.ps1`.
- Keep report files JSON-only and append new timestamped runs instead of overwriting history.
- Use the `*-latest.json` files for automation and the timestamped files for audit/history.

## Adding New Items

- Add a new report-producing suite only if it writes a stable JSON payload and a matching `*-latest.json` pointer.
- Update this README if a new report family is added.
